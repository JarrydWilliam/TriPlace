# SameVibe Founder Decisions — Refined Launch Strategy

## 1. Five-community model
* The five-active-community limit remains intact.
* A sixth join triggers an intentional replacement flow.
* The model is explained as a focus and quality mechanism.
* It is not presented only as an artificial payment restriction.

## 2. Imported events
* Original organizer must be retained.
* Original source must be retained.
* Working source URL must be retained.
* External-registration indicator must be retained.
* Attribution must be clear.
* Clear distinction from SameVibe-created activities.
* No false ownership claim.
* No fabricated member interest or attendance.

## 3. Low-density markets
* Users do not encounter dead-end experiences such as:
  > No communities found.
* The interface provides useful alternatives such as:
  * Create an activity
  * Expand the search distance
  * Explore nearby or regional content
  * Follow an interest
  * Indicate desired local content
  * Understand that SameVibe is still developing locally

## 4. Core product loop
The release must support:
1. Adult registration
2. Age eligibility
3. Interest onboarding
4. Focused community discovery
5. Community joining
6. Imported-event and user-activity discovery
7. Immediate group-activity creation
8. Shared-context communication
9. RSVP or participation
10. Post-event learning
11. Improved recommendations
12. Return participation
